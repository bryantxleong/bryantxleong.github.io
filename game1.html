About Me

“Pikachu vs Raichu” is a two-player Pong-style game.

In this game, two players compete against each other in a fast-paced match, controlling Pikachu and Raichu as paddles to hit the ball back and forth. The objective of the game is simple: the first player to reach 15 points wins the match.

This project was created as part of a school assignment and marks the first game I developed on my own. It was fully built using Stencyl, giving me the opportunity to explore the fundamentals of game development.

Challenging

1. Spike Speed Boost Conflict

In the Pong game, giving the ball a speed boost when hitting a spike is a common gameplay mechanic.

During development, although I implemented the functionality for the ball to speed up upon hitting a spike, testing the game revealed that the feature did not work as expected.

Solution

After exploring Stencyl further, I realized that to trigger a continuous event.

Such as

“the ball hits a spike → spike disappears → ball speeds up”

All related logic must be handled within the same actor.

In my original code, I coded the spike to disappear when hitting by ball, and separately coded the ball to speed up when hitting a spike. This caused a conflict, as Stencyl could not determine which action to execute first: The spike disappearing or The ball speeding up.

2. Fake Spike Horizontal Speed Bug

During the testing phase, I discovered sometimes that when the ball collided with a fake spike, its horizontal speed would instantly drop to zero.

As a result, the ball only moved up and down in place, which heavily impacted the gameplay experience.

Solution

To resolve this issue, I introduced a counter system.

When the ball’s horizontal speed became zero, I set a condition where after bouncing off the top or bottom walls twice, the ball would be assigned a random horizontal speed.

This prevented the ball from getting stuck in a vertical-only movement pattern, keeping the gameplay dynamic.
